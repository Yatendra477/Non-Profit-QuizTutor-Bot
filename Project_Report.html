Project Overview

📌 Problem Statement: Develop an interactive AI-driven educational bot for the Non-Profit domain. The bot engages users in targeted assessments based on real Donor Emails, evaluates responses using a Large Language Model, and provides deep contextual explanations for incorrect answers — creating a personalized learning loop that bridges knowledge gaps and reinforces industry-specific concepts.

The Non-Profit Quiz/Tutor Bot is a Generative AI application that transforms a curated collection of non-profit donor emails into an interactive, self-paced learning experience. Unlike traditional quiz systems that rely on static question banks with keyword matching, this bot uses Retrieval-Augmented Generation (RAG) to ground every question and explanation in real email content, ensuring factual accuracy and rich contextual depth.

When a user answers incorrectly, the bot does not just say "wrong" — it retrieves the most semantically relevant email excerpts from a vector database and asks the LLM to craft a personalised, educational explanation that references the actual email content. This creates a genuine personalized learning loop.

Key AI/ML Concepts Used

1

Large Language Model (LLM) — Google Gemini 2.5 Flash

Gemini is used for 3 tasks: (a) generating quiz questions as structured JSON, (b) semantically evaluating short-answer responses, and (c) writing contextual explanations and positive reinforcement messages.

2

Vector Embeddings & Semantic Search

Each text chunk from the donor emails is converted into a high-dimensional numerical vector using Gemini's embedding model. When the bot needs context, it finds the most semantically similar chunks using cosine similarity — not keyword matching.

3

Retrieval-Augmented Generation (RAG)

RAG combines a retrieval system (ChromaDB) with a generative model (Gemini). Instead of relying on the LLM's training data memory, the bot fetches relevant email chunks at runtime and injects them as context into the prompt. This ensures answers are grounded in actual email content.

4

Document Chunking Strategy

Long emails are split into 500-character overlapping chunks (80-char overlap) using RecursiveCharacterTextSplitter. Overlap prevents context from being cut mid-sentence, ensuring retrieval quality.

5

Prompt Engineering

Every LLM call uses carefully structured prompts with role definition, context injection, output format enforcement (JSON), and rules to ensure quality. The batch quiz prompt generates all N questions in a single API call to stay within rate limits.

RAG Explanation Workflow

💡 What is RAG?

Retrieval-Augmented Generation (RAG) grounds LLM responses in real data by retrieving relevant documents before generating a response. This prevents hallucination and makes answers factually anchored in the email knowledge base.

How the bot uses RAG for wrong-answer explanations:

1

User gives a wrong answer

The evaluator detects the answer is incorrect (direct comparison for MCQ/TF, Gemini verdict for short answer).

2

Semantic search on ChromaDB

The question text + topic are used as a search query. ChromaDB compares this query's embedding against all stored email chunk embeddings and returns the top-3 most similar chunks.

3

Context-injected explanation prompt

The retrieved email excerpts are injected into a structured prompt along with the question, correct answer, and student's wrong answer.

4

Gemini generates a grounded explanation

Gemini writes a 3–5 sentence educational explanation that references specific details from the email context — making the feedback memorable and factually accurate.

5

Source email citations displayed

The bot shows which specific donor emails (by subject line) were referenced to generate the explanation, providing full transparency.

How to Run the Bot

Install dependencies

pip install -r requirements.txt
2

Set up API key

# Copy template → .env and add your key copy .env.example .env # Edit .env → GOOGLE_API_KEY=your_key_here # Get free key at: aistudio.google.com/app/apikey

Ingest donor emails (once only)

python main.py --ingest
4

Run the quiz

python main.py # 5-question quiz (default) python main.py --quiz --num-questions 10 # 10-question quiz

python -m pytest test_smoke.py -v
